කම්පන න්‍යාය / Vibration Theory
ගණිතමය සූත්‍ර සහ භෞතික තේරුම — Mathematical formulations and physical interpretation

1. කම්පන පද්ධතිය / Vibration System

සිංහල

කම්පන පද්ධතියක් ස්කන්ධයක් (m), ඉලාස්ටික් ස්ප්‍රිං (k) සහ අවමන්දනය (c) යන කොටස් තුනෙන් සමන්විත වේ. ස්කන්ධය විස්ථාපනය වන විට, ස්ප්‍රිං පුනරාවර්තන බලයක් ඇති කරයි. කම්පනය ස්ථිර ලෙස පවතිනු ඇත්නම් කිසිදු අවමන්දන බලයක් නොතිබිය යුතුය.

English

A vibration system consists of three elements: mass (m), elastic spring (k), and damper (c). When the mass is displaced, the spring exerts a restoring force. Without damping, the oscillation continues indefinitely at the natural frequency.

2. චලන සමීකරණය / Equation of Motion

සිංහල

නිව්ටන්ගේ දෙවන නියමයට (F = ma) අනුව, සියලු බල සාරාංශ කිරීමෙන් ද්විතීයික ව්‍යුත්පන්න සමීකරණය ලැබේ:

English

Applying Newton's second law (F = ma) and summing all forces acting on the mass yields the second-order ordinary differential equation:

mẍ + cẋ + kx = F₀ sin(ωt)
Governing ODE for forced vibration with viscous damping

සිංහල — නියත අර්ථ

m = ස්කන්ධය (kg)
c = ස්නිග්ධ අවමන්දන සංගුණකය (Ns/m)
k = ස්ප්‍රිං දෘඪතාව (N/m)
F₀ = බල විස්තාරිතය (N)
ω = බල කෝණික සංඛ්‍යාතය (rad/s)

English — Variable Definitions

m = mass (kg)
c = viscous damping coefficient (Ns/m)
k = spring stiffness (N/m)
F₀ = forcing amplitude (N)
ω = forcing angular frequency (rad/s)

3. ස්වාභාවික සංඛ්‍යාතය / Natural Frequency

සිංහල

ස්වාභාවික කෝණික සංඛ්‍යාතය (ωₙ) යනු බාහිර බලයක් නොමැතිව පද්ධතිය කම්පනය වන සංඛ්‍යාතයයි. අවමන්දන සංගුණකය (ζ) 1 ට වඩා අඩු වූ විට, පද්ධතිය underamped ලෙස හැඳින්වේ.

English

The natural angular frequency (ωₙ) is the frequency at which the system oscillates freely. When the damping ratio (ζ) is less than 1, the system is underdamped and exhibits oscillatory behavior.

ωₙ = √(k/m)      ζ = c / (2√(mk))      ωd = ωₙ√(1 - ζ²)
Natural frequency | Damping ratio | Damped natural frequency

4. නිශ්චිත විසඳුම / Analytical Solution

සිංහල

Underdamped පද්ධතිය සඳහා (ζ < 1), සම්පූර්ණ විසඳුම ක්ෂය වන ස්වාභාවික (transient) ප්‍රතිචාරය සහ ස්ථිර (steady-state) ප්‍රතිචාරය ලෙස ලිවිය හැකිය.

English

For underdamped systems (ζ < 1), the complete solution is the sum of the transient response (decays with time) and the steady-state response (persists indefinitely).

x(t) = e^(-ζωₙt)[A cos(ωd t) + B sin(ωd t)] + X sin(ωt - φ)
Complete solution = Transient + Steady-state
X = (F₀/k) / √[(1-r²)² + (2ζr)²]      φ = arctan[2ζr / (1-r²)]
Steady-state amplitude X and phase angle φ, where r = ω/ωₙ

5. අනුනාදය / Resonance

සිංහල

අනුනාදය (Resonance) ඇති වන්නේ බල සංඛ්‍යාතය ස්වාභාවික සංඛ්‍යාතයට සමාන (r = 1) වූ විටය. Undamped පද්ධතිවල (ζ = 0) අනුනාදය ඇති වූ විට, විස්ථාපනය න්‍යාය ශාස්ත්‍රානුකූලව අසීමිත (infinite) ලෙස වැඩේ. ඉංජිනේරු ව්‍යූහවල මෙය ව්‍යසනකාරී විය හැකිය.

English

Resonance occurs when the forcing frequency equals the natural frequency (r = 1). For undamped systems (ζ = 0), the amplitude theoretically grows without bound. In engineering structures, this can be catastrophic — the Tacoma Narrows Bridge collapse (1940) is a classic example.

r = ω/ωₙ = 1  →  X → ∞  (when ζ = 0)
⚠ ඉංජිනේරු ව්‍යූහ නිර්මාණය කරන විට, ස්වාභාවික සංඛ්‍යාතය ක්‍රියාකාරී සංඛ්‍යාතවලින් ඈතින් තැබිය යුතුය. | In engineering design, natural frequencies must be kept away from operating frequencies to avoid resonance.

6. Euler සහ RK4 සංඛ්‍යාත්මක ක්‍රම / Numerical Methods

Euler ක්‍රමය (සිංහල)

Euler ක්‍රමය සරලම සංඛ්‍යාත්මක ඒකාබද්ධ කිරීමේ ක්‍රමයයි. එය ප්‍රථමාංශ Taylor ශ්‍රේණිය ඇසුරෙන් ගොඩනගා ඇත. Local truncation error: O(h²). Global error: O(h). dt විශාල වූ විට, Euler ක්‍රමය අස්ථාවර (unstable) විය හැකිය.

RK4 Method (English)

The 4th-order Runge-Kutta method evaluates four slope estimates per step and combines them in a weighted average. Local truncation error: O(h⁵). Global error: O(h⁴). Far more accurate than Euler for the same step size, but 4× the computations per step.

Euler:  xₙ₊₁ = xₙ + h·f(tₙ, xₙ)

RK4:   k₁ = f(tₙ, xₙ)   k₂ = f(tₙ+h/2, xₙ+h·k₁/2)
        k₃ = f(tₙ+h/2, xₙ+h·k₂/2)   k₄ = f(tₙ+h, xₙ+h·k₃)
        xₙ₊₁ = xₙ + h(k₁ + 2k₂ + 2k₃ + k₄)/6

RK4 — Global Truncation Error

O(h⁴) — The global error is proportional to the 4th power of the step size. Halving dt reduces error by a factor of 16. RK4 maintains excellent accuracy even with relatively large step sizes.

GTE ≈ C·h⁴ (where C is problem-dependent)

Euler — Global Truncation Error

O(h) — The global error is proportional to the step size. Halving dt only halves the error. Euler requires very small step sizes to match RK4 accuracy. Also prone to instability for stiff problems.

GTE ≈ C·h (much larger than RK4)

ස්ථායිතා සීමා / Stability Thresholds

සිංහල

Euler ක්‍රමය ස්ථාවර ලෙස ක්‍රියා කිරීමට, dt · ωₙ ≤ 2 නිර්ණය සපිය යුතුය. අවමන්දනය (damping) ස්ථායිතාවය වැඩිදියුණු කරයි. Stiff පද්ධතිවල Euler ක්‍රමය ඉතා කුඩා dt අගයක් ඉල්ලා සිටී.

English

For Euler method stability: the product dt · ωₙ must be ≤ 2 for undamped systems. Damping extends the stability region. For stiff problems, Euler demands very small step sizes making it computationally expensive.

Euler stable condition: dt · ωₙ ≤ 2  ⟹  dt_max = 2/ωₙ = 2√(m/k)
RK4 stable condition: dt · ωₙ ≤ 2√2  ≈  2.83

🎯 Bisection Method

Algorithm Linear Conv.

Brackets a root between [a,b] where f(a)·f(b) < 0. Repeatedly halves the interval by checking the midpoint sign. Guaranteed convergence; O(log₂((b-a)/ε)) iterations.

📐 Newton-Raphson Method

Algorithm Quadratic Conv.

Uses tangent line approximation: xₙ₊₁ = xₙ − f(xₙ)/f'(xₙ). Fastest convergence near root. Requires derivative; may diverge if x₀ is far from root.

🔄 Fixed-Point Iteration

Algorithm Linear Conv.

Rewrites f(x)=0 as x = g(x) then iterates xₙ₊₁ = g(xₙ). Converges when |g'(x)| < 1 near root. Choose g(x) carefully for convergence.

📏 Regula-Falsi (False Position)

Algorithm Superlinear Conv.

Like bisection but uses linear interpolation instead of midpoint: c = (a·f(b) − b·f(a)) / (f(b)−f(a)). Faster than bisection, guaranteed like bisection.

🔬 Müller's Method

Algorithm ≈ Quadratic Conv.

Fits a parabola through three points (x₀,x₁,x₂) and uses the quadratic formula to find the next iterate. Can find complex roots; convergence order ≈ 1.84.

Method Comparison

Bracketing Methods

Bisection & Regula-Falsi require f(a)·f(b)<0. They always converge but may be slow. Regula-Falsi uses interpolation so usually faster. Best when you can bracket the root.

Open Methods

Newton-Raphson & Fixed-Point need only one starting point. Newton is the fastest (quadratic) but needs derivative. Fixed-Point needs a clever g(x) to guarantee convergence.

Polynomial Methods

Müller's Method uses three points to fit a parabola. Nearly quadratic convergence and can find complex roots — ideal for polynomial equations.

// ============================================================
// MathSphere — Vibration Solver (C++)
// Supports: Free, Damped, and Forced vibration
// Methods: Euler (1st order) and RK4 (4th order)
// Rajarat University Research Project
// ============================================================

#include <iostream>
#include <fstream>
#include <cmath>
#include <vector>
#include <iomanip>
#include <string>

using namespace std;

// ─────────────────── System Parameters ───────────────────
struct VibParams {
    double m;     // Mass (kg)
    double k;     // Stiffness (N/m)
    double zeta;  // Damping ratio (dimensionless)
    double F0;    // Force amplitude (N)
    double omega; // Forcing frequency (rad/s)
    double x0;    // Initial displacement (m)
    double v0;    // Initial velocity (m/s)
    double dt;    // Time step (s)
    double tEnd;  // End time (s)
    string mode;  // "free" | "damped" | "forced"
};

// ─────────────────── Derived quantities ──────────────────
double naturalFreq(const VibParams& p)  { return sqrt(p.k / p.m); }
double dampingCoeff(const VibParams& p)  { return 2.0 * p.zeta * sqrt(p.m * p.k); }
double dampedFreq(const VibParams& p)    { return naturalFreq(p) * sqrt(1.0 - p.zeta*p.zeta); }
double freqRatio(const VibParams& p)     { return p.omega / naturalFreq(p); }

// ─────────────── Acceleration function a(x, v, t) ────────
double accel(double x, double v, double t, const VibParams& p) {
    double c = dampingCoeff(p);
    double F = (p.mode == "forced") ? p.F0 * sin(p.omega * t) : 0.0;
    return (F - c * v - p.k * x) / p.m;
}

// ─────────────── Analytical solution (underdamped) ───────
double analytical(double t, const VibParams& p) {
    double wn = naturalFreq(p);
    double wd = dampedFreq(p);
    double z  = p.zeta;

    // Transient: e^(-zwn*t) * [A*cos(wd*t) + B*sin(wd*t)]
    double A = p.x0;
    double B = (p.v0 + z * wn * p.x0) / wd;
    double transient = exp(-z * wn * t) * (A * cos(wd * t) + B * sin(wd * t));

    if (p.mode != "forced") return transient;

    // Steady-state
    double r   = freqRatio(p);
    double den = sqrt(pow(1.0 - r*r, 2) + pow(2.0*z*r, 2));
    double X   = (p.F0 / p.k) / den;
    double phi = atan2(2.0*z*r, 1.0 - r*r);
    return transient + X * sin(p.omega * t - phi);
}

// ─────────────────── Euler Method ────────────────────────
struct State { double x, v, t; };

State eulerStep(State s, const VibParams& p) {
    double a = accel(s.x, s.v, s.t, p);
    return {s.x + p.dt * s.v, s.v + p.dt * a, s.t + p.dt};
}

// ────────────────── RK4 Method ───────────────────────────
State rk4Step(State s, const VibParams& p) {
    double h = p.dt;
    double k1v = accel(s.x,           s.v,           s.t,       p);
    double k1x = s.v;
    double k2v = accel(s.x+h/2*k1x, s.v+h/2*k1v, s.t+h/2, p);
    double k2x = s.v + h/2 * k1v;
    double k3v = accel(s.x+h/2*k2x, s.v+h/2*k2v, s.t+h/2, p);
    double k3x = s.v + h/2 * k2v;
    double k4v = accel(s.x+h*k3x,   s.v+h*k3v,   s.t+h,   p);
    double k4x = s.v + h * k3v;
    return {
        s.x + h/6.0*(k1x+2*k2x+2*k3x+k4x),
        s.v + h/6.0*(k1v+2*k2v+2*k3v+k4v),
        s.t + h
    };
}

// ─────────── Run simulation, write CSV ───────────────────
void runSimulation(const VibParams& p, const string& filename) {
    ofstream csv(filename);
    csv << fixed << setprecision(8);
    csv << "t,euler_x,rk4_x,analytical_x,euler_err,rk4_err\n";

    State se = {p.x0, p.v0, 0.0};
    State sr = {p.x0, p.v0, 0.0};
    int steps = (int)(p.tEnd / p.dt);

    for (int i = 0; i <= steps; ++i) {
        double t  = i * p.dt;
        double xa = analytical(t, p);
        csv << t << "," << se.x << "," << sr.x << "," << xa
            << "," << fabs(se.x - xa) << "," << fabs(sr.x - xa) << "\n";
        se = eulerStep(se, p);
        sr = rk4Step(sr, p);
    }
    csv.close();
    cout << "Saved: " << filename << " (" << steps << " steps)\n";
}

// ─────────────────────── main() ──────────────────────────
int main() {
    VibParams p;
    p.m     = 10.0;
    p.k     = 100.0;
    p.zeta  = 0.05;
    p.F0    = 10.0;
    p.omega = 3.16;   // Near resonance (ωn ≈ 3.162)
    p.x0    = 0.0;
    p.v0    = 0.0;
    p.dt    = 0.01;
    p.tEnd  = 60.0;
    p.mode  = "forced";

    double wn = naturalFreq(p);
    cout << "=== MathSphere Vibration Solver ===\n";
    cout << "Natural frequency ωₙ = " << wn << " rad/s\n";
    cout << "Frequency ratio r    = " << freqRatio(p) << "\n";

    cout << "Critical dt (Euler)  = " << 2.0/wn << " s\n\n";

    p.x0    = 1.0;
    p.v0    = 0.0;
    runSimulation(p, "free_vibration.csv");

    return 0;
}

  omega = 3.16
)
wn  <- sqrt(params$k / params$m)
c   <- 2 * params$zeta * params$m * wn
wd  <- wn * sqrt(1 - params$zeta^2)
r   <- params$omega / wn

cat(sprintf("ωₙ = %.4f rad/s | ζ = %.3f | r = %.4f\n", wn, params$zeta, r))

# ─── deSolve ODE definition ───────────────────────────────
vibration_ode <- function(t, state, p) {
  x <- state["x"]; v <- state["v"]
  F <- p$F0 * sin(p$omega * t)
  dv <- (F - c * v - p$k * x) / p$m
  list(c(dx = v, dv = dv))
}

# ─── Analytical solution (forced, underdamped) ────────────
analytical_sol <- function(t, p) {
  den <- sqrt((1 - r^2)^2 + (2*p$zeta*r)^2)
  X   <- (p$F0 / p$k) / den
  phi <- atan2(2*p$zeta*r, 1 - r^2)
  A   <- -X * sin(-phi)
  B   <- (-p$zeta * wn * A - X * p$omega * cos(-phi)) / wd
  exp(-p$zeta * wn * t) * (A * cos(wd*t) + B * sin(wd*t)) +
    X * sin(p$omega * t - phi)
}

# ============================================================
# MathSphere — R Error Analysis Script
# Global & Local Truncation Error comparison
# Euler vs RK4 with convergence study
# ============================================================

library(ggplot2)
library(dplyr)

# ─── Parameters ───────────────────────────────────────────
m=10; k=100; zeta=0.05; F0=10; omega=3.0; t_end=20
wn <- sqrt(k/m); wd <- wn*sqrt(1-zeta^2); r <- omega/wn
c_damp <- 2*zeta*m*wn

# ─── Exact analytical solution ────────────────────────────
exact <- function(t) {
  den <- sqrt((1-r^2)^2 + (2*zeta*r)^2)
  X <- (F0/k)/den; phi <- atan2(2*zeta*r, 1-r^2)
  A <- -X*sin(-phi)
  B <- (-zeta*wn*A - X*omega*cos(-phi))/wd
  exp(-zeta*wn*t)*(A*cos(wd*t)+B*sin(wd*t)) + X*sin(omega*t-phi)
}

# ─── Euler solver ─────────────────────────────────────────
euler_solve <- function(dt) {
  n <- floor(t_end/dt); x <- 0; v <- 0; xs <- numeric(n+1)
  for(i in 0:n) {
    xs[i+1] <- x
    a <- (F0*sin(omega*i*dt) - c_damp*v - k*x)/m
    x <- x + dt*v; v <- v + dt*a
  }
  xs
}

# ─── RK4 solver ───────────────────────────────────────────
rk4_solve <- function(dt) {
  n <- floor(t_end/dt); x <- 0; v <- 0; xs <- numeric(n+1)
  acc <- function(xi,vi,ti) (F0*sin(omega*ti)-c_damp*vi-k*xi)/m
  for(i in 0:n) {
    t <- i*dt; xs[i+1] <- x
    k1v <- acc(x,v,t); k1x <- v
    k2v <- acc(x+dt/2*k1x,v+dt/2*k1v,t+dt/2); k2x <- v+dt/2*k1v
    k3v <- acc(x+dt/2*k2x,v+dt/2*k2v,t+dt/2); k3x <- v+dt/2*k2v
    k4v <- acc(x+dt*k3x,v+dt*k3v,t+dt); k4x <- v+dt*k3v
    x <- x+dt/6*(k1x+2*k2x+2*k3x+k4x)
    v <- v+dt/6*(k1v+2*k2v+2*k3v+k4v)
  }
  xs
}

# ─── Convergence study ────────────────────────────────────
dt_vals <- c(0.2, 0.1, 0.05, 0.02, 0.01, 0.005)
conv_df <- do.call(rbind, lapply(dt_vals, function(dt) {
  n <- floor(t_end/dt)
  ts <- (0:n)*dt
  xa <- sapply(ts, exact)
  xe <- euler_solve(dt)
  xr <- rk4_solve(dt)
  data.frame(dt=dt,
    euler_GTE = max(abs(xe - xa)),
    rk4_GTE   = max(abs(xr - xa)))
}))

cat("\n── Convergence Table ──\n")
print(conv_df, digits=4)

# ─── Log-log convergence plot ─────────────────────────────
p_conv <- ggplot(conv_df) +
  geom_line(aes(dt, euler_GTE, color="Euler O(h)"),    linewidth=1.2) +
  geom_point(aes(dt, euler_GTE, color="Euler O(h)"),   size=3) +
  geom_line(aes(dt, rk4_GTE,   color="RK4 O(h⁴)"),    linewidth=1.2) +
  geom_point(aes(dt, rk4_GTE,   color="RK4 O(h⁴)"),   size=3) +
  scale_x_log10() + scale_y_log10() +
  scale_color_manual(values=c("Euler O(h)"="#fbbf24", "RK4 O(h⁴)"="#2dd4bf")) +
  labs(title="Convergence Study — Global Truncation Error vs Step Size",
       subtitle="Log-log plot shows O(h) for Euler, O(h⁴) for RK4",
       x="Step size h (log scale)", y="Max |error| (log scale)", color="Method") +
  theme_dark() +
  theme(plot.background=element_rect(fill="#0b0f1a"),
        panel.background=element_rect(fill="#111827"),
        text=element_text(color="#e2e8f0"),
        panel.grid=element_line(color="#1a2233"),
        legend.background=element_rect(fill="#1a2233"))

ggsave("convergence_study.png", p_conv, width=10, height=5, dpi=300)
cat("Convergence plot saved!\n")
